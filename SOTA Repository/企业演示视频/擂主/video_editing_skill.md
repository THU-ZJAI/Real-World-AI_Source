# 企业级介绍视频制作标准流程

## 概述

本文档提供了一套完整的企业级介绍视频制作标准化流程，基于MCP工具链实现从原始视频素材到精简成品视频的全自动化处理。

### 核心设计原则

- **TTS时长优先**: 先生成TTS音频，视频压缩时长适配TTS
- **严格文本校对**: 确保字幕与原始文本高度一致
- **音画同步保证**: 确保音频、视频、字幕完美同步
- **用户确认机制**: 关键参数需用户确认
- **并行处理优化**: 提升整体处理效率
- **质量监控机制**: **在关键步骤增加质量检查和自动恢复功能**

## 输入要求

### 文件结构

```
项目根目录/
├── files/                          # 原始素材文件夹
│   ├── index.title.suffix          # 不同功能的录屏视频文件 (必须)
│   ├── text.txt                     # 文本素材文件 (必须)
│   └── bgm.wav                     # 背景音乐 (可选)（存在则必须用上）
└── video_editing_skill.md           # 本流程文档
```

### 文件规范

#### 文本素材文件 (files/text.txt)

- **格式**: UTF-8编码的.txt文件
- **内容结构**: 按视频序号分段，每段对应一个视频文件
- **格式要求**:

```txt
# 1
接下来我们来看一下构建流程，新建工作流，导入之前准备好的工作流，点击保存就好了。

# 2
我们来添加模型，点击左边的模型再点击右上角的添加模型。我们选择这个接口格式，这边我们用的是第三方代理，点击保存就可以了。好，到这里，模型配置就完成了。
```

**要求**:

- 文字内容与视频画面内容匹配
- 语言表达简洁流畅，适合口语化表达
- 无特殊字符和复杂格式

#### 视频文件 (files/index-title.suffix)

- **格式**: .mov, .mp4, .avi, .mkv
- **命名**: `{序号}.{描述}.mov`
- **要求**: 画面清晰，内容完整，音频清晰可识别

#### 背景音乐 (files/bgm.wav) - 可选

- **格式**: .wav, .mp3, .m4a
- **要求**: 时长 ≥ 最终视频时长，音质良好无噪音

## 技术参数

```yaml
视频处理:

  编码: UTF-8
  校对精度: 字符级匹配
  容错率: < 0.1%
```

 

### 阶段1: 环境准备

创建输出目录结构：`processed/{audio,video,srt,final}`

### 阶段2: 输入文件验证

1. 验证必需文件存在：`files/text.txt` 和视频文件
2. 解析文本素材，提取段落内容
3. 验证文本段落数与视频文件数量匹配

### 阶段3: 视频技术参数标准化

**关键重要性**: 确保所有视频段具有一致的技术参数，避免音画同步问题

#### 3.1 视频技术参数检查

1. 检查所有视频文件的技术参数：
   - 帧率 (r_frame_rate)
   - 分辨率 (width, height)
   - 编码格式 (codec)
   - 时间基 (time_base)
   - 音频采样率 (sample_rate)

2. **帧率标准化检查**：
   - 使用 `ffprobe -v quiet -select_streams v:0 -show_entries stream=r_frame_rate -of csv=p=0 [视频文件]`
   - 记录每个视频的实际帧率
   - 检查是否存在非常规帧率（如非整数帧率）

#### 3.2 异常视频标准化处理

1. **检测异常帧率视频**：
   - 识别帧率与其他视频不一致的视频文件
   - 特别关注可能的小数帧率（如56.6fps等）
   - 记录需要处理的视频文件列表

2. **帧率标准化处理**：
   - 确定标准帧率（推荐60fps，与其他视频保持一致）
   - **使用优化的ffmpeg参数进行帧率转换，保持音频同步**：
   ```bash
   ffmpeg -i [异常视频文件] -r 60 -c:v libx264 -c:a aac -async 1 [标准化后视频文件]
   ```

3. **验证处理结果**：
   - 重新检查标准化后视频的技术参数
   - 确保所有视频的帧率统一为60fps
   - **增加音频同步验证，确保帧率转换后音画同步**
   - 更新视频文件列表，使用标准化后的文件

#### 3.3 技术参数一致性报告

生成技术参数检查报告，包含：
- 原始视频技术参数列表
- 异常视频及其处理方式
- 标准化后的最终技术参数
- 质量验证结果
- **音频同步验证结果**

### 阶段4: 视频内容分析与用户确认

1. 获取所有视频的时长信息
2. 智能识别关键视频段（文件名包含"效果展示"、"demo"、"演示"等关键词或时长最长）
3. 用户确认关键参数：
    - 目标总时长（60秒/90秒/120秒）
    - 关键内容视频文件
    - 关键内容位置（开头/中间/结尾）

### 阶段5: TTS音频生成与时长计算

#### 5.1 TTS音频生成

生成自然时长的TTS音频：

1. 按段落分割文本内容
2. 逐段生成TTS音频到 `processed/audio/audio-tts-*.wav`（使用自然语速，不预设时长）
3. 获取每个TTS音频的实际时长
4. 记录文本内容与TTS实际时长的对应关系
5. **增加TTS质量检查，确保音频清晰度和自然度**

#### 5.2 视频压缩时长计算

调用"视频压缩时长计算器 - MCP工具接口"确定最终压缩参数：

1. 输入所有TTS音频的实际时长
2. 输入原始视频的时长信息
3. 根据目标总时长，智能分配各视频段的压缩时长
4. **设置最大压缩倍率限制（建议3倍），避免过度压缩**
5. 生成压缩时长配置文件，用于后续视频压缩
6. 当TTS音频的实际时长>目标时长时，**提供两个选项**：
   - 选项1：用户确认超时是否可接受
   - 选项2：**自动压缩文案并重新生成TTS**

### 阶段6: 视频/音频分离

并行执行以下任务：

1. 从原始视频中提取音频到 `processed/audio/audio-original-*.wav`
2. 提取无声视频流到 `processed/video/video-silent-*.mov`
3. **验证音频和视频流的完整性**

### 阶段7: 时长匹配压缩

基于阶段5.2计算的压缩时长配置执行视频压缩：

1. 读取压缩时长配置文件
2. 获取原始视频时长
3. 计算压缩倍率 = 原始时长 / 目标时长
4. **检查压缩倍率是否超过3倍限制，如超过则调整目标时长**
5. 应用压缩倍率压缩视频到 `processed/video/video-compressed-*.mov`
6. 验证压缩后时长与配置一致（这一步很重要，必须要调用 MCP 工具来验证！），容差在±0.1秒内
7. **增加压缩质量检查，确保视频质量在可接受范围内**

### 阶段8: 字幕生成

基于TTS音频重新生成字幕：

1. 为每个TTS音频文件生成ASR字幕
2. 保存字幕到 `processed/srt/srt-tts-*.srt`
3. 提取字幕文本内容

### 阶段9: 字幕文件逐字更新

基于files/text.txt中的标准答案文本对ASR字幕进行逐字纠错：

1. 读取files/text.txt文件中的标准答案文本内容
2. 对比ASR字幕文本与标准答案，进行智能纠错：
   - **严格保持原有的时间戳不变**
   - **严格保持原有的字幕段数和分割结构不变**
   - **使用智能纠错算法，允许字符数量的适度调整**
   - 例如：ASR字幕某段为"在毕生平台上"（6个字符），标准答案对应位置为"在毕昇平台上"（6个字符），则仅将"生"改为"昇"
   - **当字符数量不匹配时，使用智能算法调整时间戳**
   - 主要纠错类型：同音字错误、多字少字、标点符号错误等ASR识别问题
3. 生成纠错后的字幕文件到 `processed/srt/srt-final-*.srt`
4. 验证纠错后字幕与标准答案95%以上一致（忽略标点和格式差异）
5. **提供人工干预接口，允许用户手动调整纠错结果**
6. 确定最终使用的字幕文件为纠错后的版本

### 阶段10: 视频段合成

为每个视频段合成完整的音画字幕：

1. 生成静音音频作为第二音轨
2. 将压缩视频、TTS音频、字幕合成为完整视频段
3. **增加实时音画同步检查，确保合成过程无偏差**
4. 验证音画同步性（时长差异 < 0.1秒）
5. **增加字幕位置检查，确保字幕在画面合适位置**
6. 保存到 `processed/video/video-synthesized-*.mp4`
注：这一步必须调用 ffmpeg 相关 mcp 工具

### 阶段11: 视频合并

根据用户确认的关键内容位置重新排序：

1. 根据用户偏好调整视频段顺序（关键内容可置前）
2. 创建合并列表文件
3. 按顺序合并所有视频段为最终视频
4. 保存到 `processed/final/final_video_no_bgm.mp4`

### 阶段12: 背景音乐添加

如存在背景音乐文件，并行生成多个版本：

1. 无背景音乐版本
2. 5%背景音乐版本
3. 15%背景音乐版本

## 最终交付物

### 输出文件结构

```
processed/
├── final/                          # 最终视频文件
│   ├── final_video_no_bgm.mp4      # 无背景音乐版本
│   ├── final_video_with_bgm_5.mp4  # 5%背景音乐版本
│   └── final_video_with_bgm_15.mp4 # 15%背景音乐版本
├── audio/                          # 音频文件
│   ├── audio-original-*.wav        # 原始音频
│   └── audio-tts-*.wav            # TTS音频
├── video/                          # 视频文件
│   ├── video-silent-*.mov          # 无声视频
│   ├── video-compressed-*.mov      # 压缩视频
│   ├── video-synthesized-*.mp4     # 合成视频段
│   └── video-normalized-*.mov      # 标准化处理后的视频（如有）
├── srt/                            # 字幕文件
│   ├── srt-tts-*.srt              # 基于TTS生成的字幕
│   ├── srt-final-*.srt            # 验证通过的最终字幕
│   └── srt-manual-*.srt           # 人工校对的字幕
└── text/                           # 文本文件
    ├── verification_report.txt    # 文本校对报告
    ├── sync_issues.txt            # 同步问题报告
    ├── compression_config.json    # 视频压缩时长配置
    └── video_standards_report.txt # 视频技术参数标准化报告
```

### 质量标准

- **视频技术参数一致性**: 所有视频段统一为60fps，避免音画同步问题
- **总时长**: 符合确认的目标时长
- **音视频同步**: 严格同步，差异 < 0.1秒
- **文本准确性**: 与原文档95%以上相似度
- **字幕同步**: 基于TTS音频生成，时间轴精确匹配
- **关键内容**: 根据用户偏好放置（推荐开头）
- **压缩质量控制**: **最大压缩倍率不超过3倍，质量指标符合预设阈值**
- **全面验证机制**: **在关键步骤增加多维度质量检查**
